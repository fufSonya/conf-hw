## Учебный конфигурационный язык → TOML (CLI)

Реализован инструмент командной строки, который **читает входной текст из stdin** на учебном конфигурационном языке и **пишет результат в файл TOML**, путь к которому задаётся ключом командной строки.

### Что сделано по заданию
- **Поддержан синтаксис**:
  - **Числа**: `\d+`
  - **Массивы**: `#( значение, значение, ... )`
  - **Словари**:

    ```
    {
      имя => значение,
      ...
    }
    ```

  - **Имена**: `[_a-zA-Z][_a-zA-Z0-9]*`
  - **Значения**: числа / массивы / словари (в т.ч. вложенные)
  - **Объявление константы**: `значение -> имя`
  - **Использование константы**: `[имя]` (результат — значение константы)
- **Диагностика синтаксических ошибок**: при ошибках лексинга/парсинга выводится сообщение в stderr с **line/col**.
- **Тесты**: все конструкции (включая вложенность и константы) покрыты тестами.
- **3 примера конфигураций** из разных предметных областей: `examples/01_webapp.*`, `examples/02_iot.*`, `examples/03_finance.*`.

### Структура проекта
- `main.py` — CLI
- `config_translator/` — реализация транслятора (лексер/парсер/вычисление констант/генерация TOML)
- `tests/` — тесты
- `examples/` — примеры `.conf` и ожидаемый результат `.toml`

### Запуск транслятора
Linux/macOS:

```bash
python3 main.py -o out.toml < input.conf
```

Windows PowerShell:

```powershell
py -3 main.py -o out.toml < input.conf
```

Коды возврата:
- `0` — успех
- `2` — ошибка трансляции (лексическая/синтаксическая/вычисление констант)
- `3` — непредвиденная ошибка

### Примеры

```bash
python3 main.py -o /tmp/webapp.toml < examples/01_webapp.conf
python3 main.py -o /tmp/iot.toml < examples/02_iot.conf
python3 main.py -o /tmp/finance.toml < examples/03_finance.conf
```

Ожидаемые результаты лежат рядом: `examples/*.toml`.

### Запуск тестов

```bash
python3 -m unittest discover -s tests -p "test_*.py"
```

### Замечания
- Константы должны быть объявлены **до** использования (например, `10 -> x` перед `[x]`).
